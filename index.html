The definitive badge list for all of your projects.

A curated list of different Shield.io badges, organized in lists, for you to add into your documentations.

Notable Features:

Notable features that you should keep in mind before you start using this.

Simple

No bullsh*t, no extra steps, just Ctrl + C, and Ctrl + V into your file.

Open Source

This project is open source and is being maintained with new badges and updates. Feel free to submit a pull request if you want to add your own badge!

Have suggestions? Found an issue? Submit an issue!

Customization

Customizable badges offer versatile visual elements, allowing you to modify content, colors, logos, and styles to suit your needs.

Styling

Don't like a type of badge? You can style it as you see fit to change its look!

In an era of software documentation, README's and information inside of them are especially important in order to enhance user experience, or for anyone who has an issue with the software/project.

Thanks to the team behind shield.io, badges have become an integral part of modern software development, especially in open-source projects and repositories. Over the last few years, the use of badges has significantly increased due to their ability to provide quick, visual information about various aspects of a project's status, health, and functionalities.

How to Use:

There are no extra prerequisites to start using these badges. Simply copy the 'Markdown Code' of the badge that you'd like from the repository, or from GitBook, and paste it into your Markdown file.

Here's an example of a Discord badge with a link that redirects when the badge is clicked on:

README.md

[![Discord](https://img.shields.io/badge/Discord-%235865F2.svg?&logo=discord&logoColor=white)](https://discord.gg/code)

FAQ

Frequently Asked Questions

How do I use this?

Go to the GitHub repository by clicking "View On GitHub".

Look for the badge you would like to add and copy the text in the "Markdown Code" section.

Tip: Use 'Ctrl + F' or the Table Of Contents to search quickly for your badge.

Why won't my badge lead anywhere?

By default, (mostly) all badges have (#) at the end of their code to define a link.

You need to replace the (#) with a link that you'd like the badge to go to when clicked on.

For example, if the code is:

[![GitHub](https://img.shields.io/badge/GitHub-%23121011.svg?logo=github&logoColor=white)](#)

...you'll need to replace the (#) with a link (eg. https://github.com/inttter).

If the code is a dynamically updating badge such as this one:

 [![GitHub release](https://img.shields.io/github/release/badges/shields.svg)](https://github.com/badges/shields/releases)

You must replace 'badges' with your GitHub username/organization, and 'shields' with your repository name. This includes the redirect URL aswell.

Can I customize these badges?

Yes, please refer to the Styles and Customization documentations for how to style and customize your badges.

Can I use badges on any repository hosting platform?

As long as they support Markdown rendering, you can.

If you need more info, consult the help page for your platform.

Do these only work in Markdown?

No, you can also use them in HTML, by using `<a>` tags to link to a website, and `<img src>` tags to display the badge.

Here's how you may format it if you were to link the badge to the Kotlin website inside of a HTML file:

<a href="https://kotlinlang.org">
<img src="https://img.shields.io/badge/Kotlin-%237F52FF.svg?logo=kotlin&logoColor=white" alt="Kotlin">
</a>
